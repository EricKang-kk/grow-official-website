# GROW 官网决策记录

最后更新：2026-05-17

本文件用于记录项目中的重要技术、设计和协作决策。以后如果换电脑、换对话或隔一段时间继续开发，可以通过本文件快速恢复判断依据。

## 1. 决策记录格式

每条决策建议包含：

- 日期
- 决策
- 原因
- 影响
- 后续动作

## 2. 已确认决策

### 2026-05-17：使用 GitHub 作为两台 Mac 的项目同步中心

决策：

使用 GitHub 仓库同步 GROW 官网项目，而不是依赖纯本地文件夹或 iCloud 作为主要开发同步方式。

原因：

- GitHub 更适合代码项目的版本管理
- 可以在两台 Mac 之间稳定同步代码
- 可以记录历史版本，方便回滚
- 后续接入 Vercel 部署更顺畅
- Codex 在不同电脑上可以通过读取项目文件恢复上下文

影响：

- 每台电脑都需要 clone 仓库
- 每次换电脑开发前需要先 git pull
- 每次完成阶段性改动后需要 commit 并 push

后续动作：

- 保持重要进度写入 ROADMAP.md 和 CHANGELOG.md
- 保持重要取舍写入 DECISIONS.md

### 2026-05-17：两台 Mac 分别配置独立 SSH Key

决策：

每台 Mac 使用独立的 GitHub SSH Key。

原因：

- 每台设备都有独立授权，更清晰也更安全
- 如果某台设备不再使用，可以只删除对应 key
- 避免在不同电脑之间复制私钥

影响：

- 新设备第一次使用时需要生成 SSH Key 并添加到 GitHub
- GitHub 账号设置中会看到不同设备的 key，例如 MacBook Codex、Mac mini Codex

后续动作：

- 不共享私钥
- 只把公钥添加到 GitHub

### 2026-05-17：用项目文件沉淀跨对话上下文

决策：

不依赖 Codex 对话本身同步上下文，而是将长期上下文写入项目文件。

原因：

- 不同电脑上的 Codex 对话无法自动共享聊天记录
- 项目文件可以通过 GitHub 同步
- 新对话可以通过读取项目文件恢复背景、进度和决策

影响：

- 每个阶段完成后，应更新 ROADMAP.md、DECISIONS.md 或 CHANGELOG.md
- 新对话开始时，应要求 Codex 先阅读这些文件

后续动作：

- 每个模块完成后更新 CHANGELOG.md
- 路线或优先级变化时更新 ROADMAP.md
- 重要技术或设计取舍变化时更新 DECISIONS.md

### 2026-05-17：采用 Next.js + TypeScript + Tailwind CSS

决策：

官网技术栈采用 Next.js、TypeScript 和 Tailwind CSS。

原因：

- Next.js 适合构建官网和部署到 Vercel
- TypeScript 有利于减少初学阶段的低级错误
- Tailwind CSS 适合快速落地统一视觉规范
- 生态成熟，Codex 支持度高

影响：

- 项目会使用 App Router 结构
- 样式主要通过 Tailwind 实现
- 组件需要保持清晰、易读，避免过度抽象

后续动作：

- 初始化项目时选择 TypeScript 和 Tailwind CSS
- 尽量使用简单组件拆分

### 2026-05-17：优先服务初学者学习和长期维护

决策：

项目实现优先清晰、稳定、可理解，而不是追求复杂架构或炫技效果。

原因：

- 项目所有者是编程初学者
- 目标是边开发边学习真实项目流程
- 代码需要便于以后自己修改和维护

影响：

- 避免过度抽象
- 避免引入不必要的库
- 组件命名要直观
- 数据结构要容易阅读

后续动作：

- 每次开发完成后说明改动文件
- 对复杂逻辑保留必要的简短注释

### 2026-05-17：视觉风格保持温暖、明亮、圆润

决策：

GROW 官网采用温暖、明亮、圆润、插画感的视觉方向，避免冷硬商业风。

原因：

- GROW 面向大学生和青年群体
- 团队定位是成长支持和长期陪伴
- 网站需要建立亲近感和可信度

影响：

- 页面底色优先使用 #F9FBF7 或白色
- 主色使用绿色体系
- 多用留白、圆角和轻量插画感元素
- 避免暗黑、压迫、传统机构招生页风格

后续动作：

- 设计系统阶段将品牌色写入 Tailwind 配置
- 首页和组件开发时持续检查视觉气质

- 品牌视觉方向变化
- 部署方式变化
- 表单、数据、第三方服务方案变化
- 任何会影响后续开发方式的重要选择
